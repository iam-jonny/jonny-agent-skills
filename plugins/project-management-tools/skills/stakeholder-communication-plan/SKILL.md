# Stakeholder Communication Plan

## Shared Reference

Use `../../references/project-management-principles.md` for general project
management principles. Apply stakeholder communication, status reporting, risk,
and change control sections.

## Communication Stance

Communication should reduce surprise and speed decisions. Different
stakeholders need different detail, cadence, and asks. Do not send the same
status narrative to everyone by default.

## Workflow

1. Identify sponsor, decision owner, approvers, contributors, consulted
   parties, informed parties, and likely blockers.
2. Determine what each group needs to know, decide, provide, or monitor.
3. Define cadence, channel, format, and owner for each communication.
4. Create escalation path for risks, blockers, and change requests.
5. Draft reusable update and decision-request templates.

## Output

Return:

- Stakeholder map.
- Communication matrix.
- Decision and escalation path.
- Cadence and channels.
- Message themes by audience.
- Status update template.
- Decision request template.
- Risk escalation template.
- Open stakeholder questions.

## Common Failure Modes

- Treating stakeholders as one audience.
- Updating people without asking for needed decisions.
- No escalation route for blockers.
- Sponsor learns about risk too late.
- Communication plan does not match project governance.

## Quality Bar

The plan should tell the project team who needs what information, when, through
which channel, and for what decision or action.
