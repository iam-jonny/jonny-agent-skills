# Project Plan Create

## Shared Reference

Use `../../references/project-management-principles.md` for general project
management principles. Apply planning, risk, stakeholder communication, status
reporting, and change control sections.

## Planning Stance

Make the plan useful for coordination, not just documentation. Show milestones,
owners, dependencies, risks, decision points, and communication cadence. Avoid
fake certainty when details are unknown.

## Workflow

1. Confirm project objective, scope, constraints, timeline expectations, and
   planning horizon.
2. Break work into milestones, workstreams, tasks, owners, and dependencies.
3. Identify decision points, assumptions, risks, and escalation triggers.
4. Define communication cadence and status reporting expectations.
5. Call out missing information that blocks credible planning.

## Output

Return:

- Planning assumptions.
- Milestone plan.
- Workstreams and owners.
- Task breakdown.
- Dependencies.
- Risks and mitigations.
- Decision points.
- Communication and status cadence.
- Change control expectations.
- Open questions.
- Immediate next actions.

## Common Failure Modes

- A task list with no owners or dependencies.
- Milestones that are dates but not meaningful outcomes.
- Hidden assumptions about capacity or stakeholder availability.
- No escalation path for risks or blockers.
- No mechanism for scope or schedule changes.

## Quality Bar

The plan should help a team start execution and know what to do when reality
deviates from the plan.
