# Context Discovery

## Shared Reference

Use `../../references/meeting-intake-principles.md` for minimal questions,
context discovery, and routing guidance.

## Discovery Stance

Ask only for context that materially changes the output. Prefer candidate
selection over open-ended interrogation.

## Workflow

1. Identify missing context: product/project, initiative, stakeholders, target
   artifact, repository, output destination, or decision state.
2. Distinguish blocker context from useful-but-optional context.
3. Propose likely context candidates when available.
4. Ask concise user questions.

## Output

Return:

- Missing context.
- Blocker questions.
- Optional context questions.
- Suggested local paths or repositories to inspect.
- Recommended next skill.

## Quality Bar

Keep questions few. If a reasonable assumption can be made safely, state it and
continue.
