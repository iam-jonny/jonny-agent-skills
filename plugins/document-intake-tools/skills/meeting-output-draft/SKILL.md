# Meeting Output Draft

## Shared Reference

Use `../../references/meeting-intake-principles.md` for meeting types, output
modes, routing, and safety guidance.

## Drafting Stance

Create an output that fits the destination. Slack should be concise and action
oriented. Documentation outputs can be more structured. Preserve decisions,
owners, dates, risks, and open questions.

## Destination Formats

- Slack: concise summary, decisions, actions, risks, asks.
- Email: subject, executive summary, decisions, actions, next steps.
- Markdown/Notion/Confluence: structured sections and durable decisions.
- Word outline: formal document sections.
- PowerPoint outline: slide titles and key bullets.

## Output

Return:

- Destination-specific draft.
- Decisions made.
- Action items.
- Risks and blockers.
- Open questions.
- Suggested artifact updates.
- Follow-up message or posting note.

## Quality Bar

Do not auto-finalize sensitive content. If the output is intended for Slack or
email, make uncertainty and approval status explicit.
